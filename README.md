# HBnB Evolution - Technical Documentation

## Part 1: Architecture and Design

This document provides the technical architecture and design specifications for the HBnB Evolution application, a simplified AirBnB-like platform.

## Author
**Norah Alnujidi**

## Project Overview

HBnB Evolution is a web application that enables users to list properties, write reviews, and manage amenities through a three-layered architecture.

## Contents

1. **High-Level Architecture**
   - Three-layer architecture (Presentation, Business Logic, Persistence)
   - Facade pattern implementation
   - Package diagram

2. **Business Logic Layer**
   - Detailed class diagram
   - Entity relationships (User, Place, Review, Amenity)
   - BaseModel implementation

3. **API Interaction Flow**
   - User Registration sequence
   - Place Creation sequence
   - Review Submission sequence
   - Fetching Places sequence

## Core Features

- **User Management**: Registration, authentication, and profile management
- **Place Management**: Property listings with location and pricing
- **Review Management**: User reviews and ratings
- **Amenity Management**: Property features and facilities

## Technical Stack

- Three-layered architecture
- Facade design pattern
- RESTful API design
- UML diagrams for documentation
